## Findings Summary

**Severity Breakdown:** 2 Critical, 3 High, 3 Medium, 0 Low

### Critical Issues
1. **Admin secret hardcoded as default** — `process.env.ADMIN_SECRET || "slotted-admin-2026"` at line 9337; allows unauthorized admin endpoint access if env var unset
2. **OAuth tokens stored in plaintext** — Google, Apple, Outlook tokens as plaintext TEXT in users table; database breach exposes calendar access

### High Issues
1. **OAuth callback has no CSRF protection** — `GET /calendar/callback` and `/calendar/outlook/callback` use bare Firebase UID as `state` parameter; attacker can associate their tokens with victim's profile
2. **Apple CalDAV credentials stored in plaintext** — `apple_caldav_password` unencrypted; high-value target combined with service-role bypass
3. **In-memory rate limiter resets on cold start** — Maps per-instance, attackers can bypass by distributing across instances or waiting for cold start

### Medium Issues
- Suggestion friendId not validated as friend
- `getDbUser` returns `select("*")` including all sensitive tokens
- Plus additional findings on rate limiting and API design
