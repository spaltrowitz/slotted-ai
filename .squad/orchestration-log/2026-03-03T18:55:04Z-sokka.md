## Findings

### HIGH-Priority Bugs (in production)

1. **HIGH-1: Creator Time Change Silently Overrides Group Meetup**
   - Problem: Creator drags event in Google Calendar → all participants' times silently change with no consent
   - Scope: Group meetups (3+ participants) only; 1:1 meetups arguably fine
   - Fix: Route group meetup creator changes through counter-propose flow
   - Effort: ~2 hours
   - **Recommendation:** Fix BEFORE Phase 4

2. **HIGH-2: 410 Stale Sync Token Doesn't Retry in Webhook Handler**
   - Problem: Webhook receives 410 → token cleared → exits without retry → one-webhook delay before catch-up
   - Scope: Edge case, self-healing but adds invisible latency
   - Fix: Add retry block after clearing token
   - Effort: ~30 minutes
   - **Recommendation:** Fix BEFORE Phase 4 (bundle with HIGH-1)

### Phase 4 Recommended Ordering

1. **Integration tests** (FIRST — 2–3 days) — Zero tests exist. Every past bug found by code review. Build minimum harness for webhook → sync → RSVP flow before adding more code.
2. **Structured logging/monitoring** (SECOND — 1 day) — No visibility into sync behavior. Add queryable log lines to webhook and sync paths.
3. **Rate limiting / webhook debounce** (THIRD — half day) — Per-user debounce + 429 backoff. Low risk at ~20 users but good preventative investment.
4. **Apple Calendar CalDAV** (DEFERRED) — No user demand, high complexity, Google sync not hardened yet. Defer to Phase 5.
