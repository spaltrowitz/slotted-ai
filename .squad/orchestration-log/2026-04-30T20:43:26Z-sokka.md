## Findings Summary

**Severity Breakdown:** 4 Critical, 7 High, 9 Medium, 5 Low

### Critical Issues (Ship-Blocking)
1. **Outlook tokens NOT in SENSITIVE_FIELDS** — `outlook_access_token`, `outlook_refresh_token`, `outlook_token_expires_at` leak to client via `GET /users/me`
2. **No account deletion endpoint** — GDPR/privacy violation; no `DELETE /users/me` exists; Apple App Store and GDPR require this
3. **Admin secret hardcoded fallback** — `ADMIN_SECRET || "slotted-admin-2026"` at line 9337; unset env var allows unauthorized admin endpoint access
4. **Friend list leaks email addresses** — `GET /friends` response includes `friend.email` for every friendship; combined with invite system enables email harvesting

### High Issues
1. No input length validation on any text field — 10MB title possible, causes DB bloat and OOM on notifications
2. `GET /friends` returns `select("*")` on joined users — all columns including tokens in memory
3. Race condition in friend request upsert — declined status can be overwritten to `pending` via re-send
4. Availability overlap syncs friend's calendar without consent — potentially expensive and privacy-invasive
5. `parseInt(travelBuffer, 10)` with no range validation — can set to 99999 or negative, breaks availability
6. `zonedToUtc` timezone helper approximate — DST edge cases cause off-by-hour errors
7. No friendship verification on `POST /meetups/:meetupId/counter-propose` — deleted friends retain interaction rights
